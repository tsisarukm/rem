# Проект "Хронорус": RAG-система по истории России

## Описание проекта
"Хронорус" — это LLM-based RAG-система, специализирующаяся на истории России. Главная идея данного проекта - сделать "надежный" RAG инструмент: хочется избежать галлюцинаций по вопросам дат, имен, событий.

Бот строго опирается на загруженную энциклопедическую базу и использует гибридный поиск (семантика + точный поиск по ключевым словам и датам), чтобы предоставлять максимально достоверные факты (и побыстрее). Возможно использование не гибридного а каскадного поиска (не параллельно, а последовательно), но тут надо смотреть будет на функционал фрэймворков.
Проект будет реализован в виде сайта, но локального (ну мы мопы), типо http://localhost... На сайте должен быть относительно симпатичный чат-бот.

## Архитектура и стек (подробнее в описании архитектуры)
1. **Векторная БД:** Qdrant (выбрана из-за нативной поддержки Hybrid Search и удобной Python-библиотеки + у нее отличная дока, были примеры на семинаре). Запускается локально через Docker.
2. **Embeddings:** Легкие модели с HF (`sergeyzh/BERTA`), рассчитываются на CPU локально или на GPU в коллабе/каггле (один раз).
3. **LLM:** Бесплатные с OpenRouter API (использование inference API для избежания необходимости в локальных GPU). Возмжно несколько моделей на случай падения одной из них.
4. **Backend/Бизнес-логика:** FastAPI + чистый Python API-wrapper (без абстракций типо LangChain для полного контроля над промптами и пайплайном, на семинаре понятно было, что там много нюансов и потенциальных багов).
5. **Frontend:** Streamlit (за фронтэнд шарит примерно никто, возможно что-то поменяется, пока выглядит как оптимальное решение в нашей ситуации)

## Данные
Для базы знаний думаем использовать парсинг исторических статей из русскоязычной Википедии и открытая историческая литература (учебные пособия, учебники). Текстовые чанки предварительно обрабатываются: из них извлекаются важные метаданные (например, явно упомянутые века и конкретные года) для улучшения поиска.

## Потенциальные улучшения/изменения
Пока трудно прикинуть объем текстов и работы, так что можем менять объем данных (например, История России до 19 века только). Если же захочется наоборот улучшить и нарастить объем (в чем я сомневаюсь), то можно, например, научить модель создавать тесты или вообще применить знания с курса LlmSec и защитить нашу систему от промт-инъекций и джейлбрейков.
